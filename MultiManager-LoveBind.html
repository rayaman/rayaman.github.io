Each object has its purpose and their overall toll on the system. I use the terms Runners, Actors, Semi-Actors, and Non-Actors. Runners use a small percentage of the processing power within MultiManager library, however Runners can contain Actors and Semi-Actors, so as those are added the total load increases on Runners. Actors are tested/acted upon every step. What are steps? Steps are each loop that the library goes through. think of a for loop for a second. "for i=1,10 do multi:uManager() end" In this case there would be 10 steps that the program went through. Semi-Actors come in different forms, they tend to use a lot less processing because the function overhead is lower. Non-Actors are not part of the loop so they don't take up any. I can create hundreds of a certain non-actor and see no preformance drop. Well as long as I don't trigger them.


So what objects are: Runners, Actors, Semi-Actors and Non-Actors?


Runners

Right now we have 2 (technically 3) Runners:

  1. intObj=multi:newInterface([string: FilePath defualt: nil])
  2. intObj=multi:newQueue([string: FilePath defualt: nil])


The "multi" object is the third runner and is the main one that is used. There are some features that do not work on this object. Pause(), Resume() and Destroy() do not work with the 'multi' object.

For the first 2 interface Objects you don't need to input a filepath, however loading an interface/queue from an external file keeps the main code clean and less cluttered.

Check the examples page for how to do this.


Interfaces work just like the 'multi' Object, they use inheritance and take all the methods and variables from the multiobject. The only difference is interfaces and all the objects they contain cand be paused where the 'multi' object can not be paused.


Queues work like the interface but they do something different. Instead of each object running side by side like normally do they run one at a time in that queue. However other objects running in a seperate interface/queue will still run side by side. The examples on the examples page will show this.


Actors

Right now we have 9 Actors:

  1. eventObj=multi:newEvent([function: TASK defualt: function() end])
  2. alarmObj=multi:newAlarm([number: SET defualt: 0])
  3. loopObj=multi:newLoop([function: FUNC])
  4. stepObj=multi:newStep([number: START defualt: 0],[number: RESET defualt: inf],[number: COUNT defualt: 1],[number: SKIP defualt: 0])
  5. tstepObj=multi:newTStep([number: START defualt: 0],[number: RESET defualt: inf],[number: COUNT defualt: 1],[number: SET defualt: 1])
  6. updaterObj=multi:newUpdater([number: SKIP defualt: 0])
  7. watcherObj=multi:newWatcher(table: NAMESPACE,string: NAME)
  8. multiObj=multi:newCustomObject([table: OBJREF],[string: T='process'])
  9. void=multi:newThread(string: name,function: func)
